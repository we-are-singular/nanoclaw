# Read-Only SQLite UI — Design

**Date:** 2026-02-28

## Overview

A lightweight read-only web UI for inspecting the NanoClaw SQLite database. Starts automatically with the nanoclaw service. No editing — view only.

## Architecture

A new `src/ui-server.ts` spins up a Fastify HTTP server alongside the main nanoclaw process. It reads from the same SQLite DB using existing `db.ts` functions. Port is configurable via `UI_PORT` env var, defaulting to `3001`.

## API Routes

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | Serves the HTML shell |
| GET | `/api/groups` | Registered groups joined with last message time from `chats` |
| GET | `/api/tasks` | All scheduled tasks |
| GET | `/api/messages/:jid` | Latest 50 messages for a thread |

## Frontend

Single HTML string served at `GET /`. Vanilla JS, no build step, no framework.

**Local store pattern:**
- `store = { groups, tasks, messages, selectedJid }` object holds current state
- `setInterval(refresh, 5000)` refetches all data every 5 seconds
- Each refresh diffs against the store and re-renders only changed sections
- "Last updated: Xs ago" indicator in corner

**Three tabs:**
- **Threads** — table of registered groups: name, folder, channel, last activity. Click a row to drill into messages.
- **Schedules** — table of scheduled tasks: group folder, schedule type/value, status, next run, prompt (truncated, expands on click).
- **Messages** — message list for the selected thread, shown below/alongside the Threads tab when a row is clicked. Shows sender, timestamp, content.
